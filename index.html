Monke PDF User Manual

1. How It Works

Monke PDF uses a local Retrieval-Augmented Generation (RAG) architecture. When you upload a PDF, it parses it into sections, generates math vectors representing context (embeddings), and indexes them. When you ask a question, it finds the relevant parts of the document and passes them as context to a local LLM.

2. Supported Formats & Limits

  • File Type: Portable Document Format (.pdf) only. Scanned images or digital layouts.
  • File Size Limit: Up to 50 MB.
  • Auto-OCR: Scanned PDFs that contain non-selectable text are automatically parsed page-by-page using local neural OCR.

3. Understanding Answering Modes

  • Strict Mode (Doc-Only): The chatbot relies *solely* on the text inside the PDF. If your question is not answered in the document, it will refuse to guess, avoiding AI hallucinations. Best for contract reviews, audits, and research papers.
  • Open Mode (Assistant): The chatbot uses the PDF context as a baseline, but is allowed to utilize its general background intelligence to explain definitions, answer general follow-ups, or expand on topics.

4. Conversation Memory

The chatbot retains conversational history, letting you ask follow-up questions naturally (e.g., "What is project X?" followed by "Can you list its three core requirements?").

5. Storage & Privacy

To avoid reprocessing, the backend automatically caches the last 5 PDFs processed. If you reload a cached PDF, it will open instantly. Cache data is saved fully offline in your project folder.
